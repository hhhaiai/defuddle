# website-free 百度图片托管与详情页按钮重构设计

## 背景
用户希望 `website-free` 的详情页在浏览器查看器中：

1. 左侧 Markdown 显示区 `#md-input` 右上角提供一键复制按钮。
2. 右侧渲染区 `.preview-pane` 右上角提供“保存成 Word”按钮。
3. 页面中出现的图片统一改为上传到百度图床后的地址。
4. 用户已明确选择范围 B：不仅右侧预览，左侧 Markdown、`.md` 下载、`?raw=1` 返回也要统一替换为百度地址。

## 目标
在不新增第三方依赖的前提下：

- 将正文 HTML 中的外链图片 URL 上传到百度并替换；
- 让 Markdown 输出、viewer 预览、Word 导出三者保持同一套图片地址；
- 调整详情页按钮布局，满足左右分栏各自操作；
- 提供可复现的本地回归脚本，验证图片替换与按钮布局。

## 方案
### 图片替换时机
在 Worker 服务端、HTML 转 Markdown 之前处理正文 HTML：

- 解析正文 HTML 中的 `<img src>`；
- 对每个外链图片执行百度上传；
- 成功时替换为百度返回 URL；
- 失败时保留原图地址，不阻塞整页返回。

这样后续 Markdown 转换、viewer 渲染、下载导出都会自动使用新地址。

### 百度上传实现
在 `website-free/src` 内新增 TypeScript 工具模块，按用户提供的 Python 逻辑实现：

- 下载原始图片二进制；
- 推断 MIME；
- 转成 `data:mime;base64,...`；
- 使用 MD5 生成 token；
- POST 到百度接口；
- 读取返回 JSON 中的 `data.url`。

默认调用真实百度地址；测试时允许通过环境变量改为本地 mock endpoint。

### 按钮布局
将现有顶部全局 toolbar 改为分栏按钮：

- 左侧 `pane-header`：标题 + 复制按钮；
- 右侧 `pane-header`：标题 + 保存成 Word 按钮；
- 原全局 `Save .md` 按钮移除，保留页面主要操作聚焦在用户指定的两个按钮。

### Word 导出
保留前端导出，但改成直接下载兼容 Word 的 `.doc` 文件，而不是仅打开打印窗口。

## 风险
- 百度上传是外部依赖，偶发失败时必须降级保留原图。
- 首次请求会因为图片上传变慢，但页面最终响应应保持可用。
- Worker 运行时不能直接执行 Python，因此只能做等价 TypeScript 实现，而不是运行 `v2.py`。
